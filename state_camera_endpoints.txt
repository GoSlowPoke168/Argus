United States DOT Traffic Camera API Endpoints & Access Guide
=============================================================

This document contains a compiled list of known live traffic camera API endpoints for various US states. Due to the decentralized nature of state DOTs, there is no single unified standard, but many states share common software platforms (like Castle Rock OneWeb/511).

-------------------------------------------------------------
1. THE "511 V2" STANDARD (CASTLE ROCK / ONEWEB)
-------------------------------------------------------------
Many states use this platform. The endpoints often require a free developer token from the state's 511 website. The structure is generally: https://[state511website]/api/v2/get/cameras

*   New York: https://511ny.org/api/v2/get/cameras
*   Georgia: https://511ga.org/api/v2/get/cameras
*   Wisconsin: https://511wi.gov/api/v2/get/cameras
*   Arizona: https://az511.com/api/v2/get/cameras
*   Idaho: https://511.idaho.gov/api/v2/get/cameras
*   Iowa: https://511ia.org/api/graphql (GraphQL) OR https://tr.511ia.org/tgcameras/api/cameras
*   Minnesota: https://511mn.org/api/v2/get/cameras
*   Nevada: https://nvroads.com/api/v2/get/cameras
*   Virginia: https://511virginia.org/api/v2/get/cameras

-------------------------------------------------------------
2. SPECIFIC STATE REST & ARCGIS ENDPOINTS
-------------------------------------------------------------

*   IOWA (ArcGIS Feature Server - No Token Needed):
    Endpoint: https://services.arcgis.com/8lRhdTsQyJpO52F1/ArcGIS/rest/services/Traffic_Cameras_View/FeatureServer/0/query?where=1%3D1&outFields=*&f=pjson
    Format: GeoJSON/Esri JSON

*   CALIFORNIA (Caltrans CWWP2 - No Token Needed):
    Endpoint Format: https://cwwp2.dot.ca.gov/data/d{DISTRICT_NUMBER}/cctv/cctv.json
    Examples:
    District 11 (San Diego): https://cwwp2.dot.ca.gov/data/d11/cctv/cctv.json
    District 4 (Bay Area): https://cwwp2.dot.ca.gov/data/d4/cctv/cctv.json
    Format: JSON Array

*   WASHINGTON (WSDOT):
    Endpoint: https://wsdot.wa.gov/Traffic/api/HighwayCameras/HighwayCamerasREST.svc/GetCamerasAsJson
    Format: JSON Array (Often requires a free API code appended as ?AccessCode={code})

*   OHIO (ODOT OHGO):
    Endpoint: https://api.ohgo.com/api/v1/cameras
    Format: JSON

*   OREGON (TripCheck):
    Endpoint: https://api.odot.state.or.us/tripcheck/Cctv/Inventory
    Note: Requires an Ocp-Apim-Subscription-Key in headers.

*   PENNSYLVANIA (PennDOT):
    Various internal endpoints. Many developers use community wrappers like:
    https://github.com/chap19150/Penndot-Traffic-Cameras-API

-------------------------------------------------------------
3. MAJOR NATIONAL AGGREGATORS (EASIEST APPROACH FOR ALL 50 STATES)
-------------------------------------------------------------
If you want to pull all 50 states without managing 50 different APIs, you should use these community or commercial aggregators:

*   Road511 API (40+ States):
    Endpoint: https://api.road511.com/api/v1/features?type=cameras&jurisdiction={STATE_ABBREVIATION}
    Example (CA): https://api.road511.com/api/v1/features?type=cameras&jurisdiction=CA

*   OpenTrafficCamMap (GitHub JSON DB):
    Database: https://github.com/AidanWelch/OpenTrafficCamMap
    Contains a crowdsourced JSON database of 7,500+ global cameras.
